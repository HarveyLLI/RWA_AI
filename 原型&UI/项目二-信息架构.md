# 项目二：基金会发行产品站 — 信息架构

> 整理日期：2026-02-26  
> 用途：项目二独立网站导航与页面结构；与项目一（Antalpha 官网）区分，以基金会发行产品与投资者服务为主导。

---

## 一、站点定位

- **主体**：基金会（发行方）独立网站。
- **主要用户**：投资者（参与 Earn 产品）、AI Agent 运营方（API Key）、需要帮助的用户（AI 客服）。
- **核心能力**：产品浏览与投资、连接钱包、我的持仓、AI Agent API Key 创建与管理、AI 客服。

---

## 二、主导航结构

| 导航项 | 说明 | 对应内容/页面 |
|--------|------|----------------|
| **产品** | 基金会发行的 Earn 产品列表，按类型筛选 | 产品列表页（含卡片：名称、APY、期限、抵押/托管、CTA 参与） |
| **我的持仓** | 当前钱包下的投资头寸与收益 | 持仓列表、各产品份额、累计收益、赎回入口（需已连接钱包） |
| **API Keys** | 为 AI Agent 创建与管理 API 密钥 | 创建 Key、列表（名称、权限、创建时间）、复制/撤销 |
| **帮助** | 文档、FAQ、透明与安全说明 | 帮助中心/文档入口、常见问题、Powered by Antalpha Prime、审计与合规 |
| **连接钱包** | 全局操作，非导航页 | 未连接时显示「连接钱包」按钮；已连接显示地址缩写与断开 |

可选扩展：**关于我们** / **合规与风险**（可并入帮助或独立 tab）。

---

## 三、页面与区块

### 3.1 产品（首页 / 产品列表）

- **首屏**：站点名称 + 一句话价值主张 + 核心数据（如 TVL、产品数、累计为投资者创造收益）+ CTA「查看产品」。
- **产品列表**：卡片展示各 Earn 产品（质押借贷、矿机贷、电费贷、多资产池等），每卡含：产品名、类型、APY、期限、抵押/托管简述、「查看详情」/「参与」。
- **筛选/分类**：按产品类型、收益区间、风险标签筛选（原型可先做 Tab 或下拉）。

### 3.2 我的持仓

- **前置条件**：已连接钱包；未连接时提示先连接钱包。
- **内容**：当前地址在各产品中的份额、预估收益、可赎回金额、操作按钮（赎回等）。
- **汇总**：总持仓价值、累计收益（可选）。

### 3.3 API Keys（AI Agent）

- **入口**：导航「API Keys」或「开发者 / AI Agent」。
- **创建**：表单：Key 名称、权限范围（只读/交易等）、过期时间（可选）；提交后展示一次完整 Key（提醒妥善保存）。
- **管理**：列表展示已创建 Key（名称、权限、创建时间、最后使用）；支持复制、撤销（Revoke）。
- **说明**：简短文案说明 Key 用于 AI Agent 调用接口、数据范围与合规要求。

### 3.4 帮助

- **文档**：链接至产品说明、参与流程、赎回规则、API 文档（可与项目一文档站区分：项目二偏投资者与 Key 管理）。
- **FAQ**：常见问题（如何参与、如何赎回、KYC 要求、Key 权限等）。
- **透明与安全**：链上可查、审计报告、Powered by Antalpha Prime、托管与风控简述。

### 3.5 全局能力（各页共用）

- **连接钱包**：Header 内按钮；点击后弹出钱包选择（原型可 Mock：MetaMask / WalletConnect / 已连接 0x…ab），连接成功后显示地址缩写与「断开」。
- **AI 客服**：右下角常驻入口（图标 + 「AI 客服」）；点击展开聊天面板，支持文本输入与模拟回复（原型可固定几条回复），可收起/展开。

## 五、与项目一的衔接

- **Powered by Antalpha Prime**：在帮助或页脚标明技术与风控由 Antalpha Prime 提供，可链回项目一官网。
- **API 文档**：API Key 可注明「调用规范见 Antalpha Prime 文档站」，避免重复维护。
- **设计统一**：色调与组件风格与项目一保持一致（深蓝/浅蓝/白、Inter、机构感）。
