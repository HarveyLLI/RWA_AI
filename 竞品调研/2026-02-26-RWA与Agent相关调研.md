## 一、RWA / DeFi / Prime 类平台 — 业务呈现与透明度设计

### 1.1 RWA.xyz（行业级透明度看板）

- **定位**：Tokenized RWA 的行业标准透明度与数据分析平台，被机构、监管与投资者广泛引用。
- **数据呈现**：
  - **资产分类**：Government Securities、Stablecoins、Commodities & Precious Metals、Stocks、Global Bonds、Institutional Funds、Real Estate。
  - **实时市场数据**：TVL、24h 变化、涨跌幅；网络维度覆盖 26+ 链（Ethereum 占约 57% RWA 市场份额）。
  - **核心指标**：Total RWA value、holder 数量、稳定币市值等；支持 API 与数据下载。
- **机构背书**：Moody's、Coinbase、Bloomberg、NASDAQ、Forbes 等引用或背书，强调**机构可信度**。
- **对我们的启示**：官网/新导航下的「RWA 风控与透明度看板」可参考其**全局指标 + 分类 + 多链/多资产**的呈现方式；强调**第三方引用与背书**以增强信任。

### 1.2 RWA Pipe（开发者与 AI 数据基础设施）

- **定位**：Real World Asset Data for **Developers & AI**；为开发者与 AI Agent 提供链上 RWA 数据。
- **产品结构**（可直接对标我们的「技术能力 + Agent」呈现）：
  - **REST API**：tokens、TVL、transfers、holders、market data 等 JSON 端点。
  - **MCP Server**：Connect Claude、GPT 或 AI agents to **live RWA data**；文档中明确「Built for developers, AI agents, and DeFi protocols」。
  - **Risk API**：7-factor risk scores for DeFi collateral decisions（风险评分，可与我们的风控透明度结合）。
  - **Vault**：去中心化文档存储与公证（Beta，Sui）。
- **首页呈现**：Live tracking $330B+ tokenized assets；Four products（API、MCP、Risk、Vault）清晰分块；Daily Movers、Top RWA Tokens 表格；**Multiple ways to integrate**（REST、MCP、Webhooks）并附代码示例（curl、claude_desktop_config.json）。
- **对我们的启示**：官网「开发者与 AI-Agent 门户」可参考其**API + MCP + Risk** 并列呈现、**代码块 + 文档链接**的布局；首页即可传达「为开发者与 AI 提供数据」的定位。

### 1.3 Centrifuge Prime（机构级 RWA 平台）

- **定位**：机构级 RWA 代币化与管理；面向 DeFi 协议与 DAO 的**投资组合定制**。
- **设计要点**：
  - **组合定制**：用户可按资产类别配置（US treasuries、investment-grade credit、ABS、房地产融资等）。
  - **一体化基础设施**：代币化、证券化、法律框架、信用分析、资产接入集中在一个界面。
  - **价值主张**：Yield generation、diversification、uncorrelated assets；与 Aave、Maker、Circle 等集成。
- **对我们的启示**：官网新导航下可强调「技术 + 合规 + 资产接入」的一体化能力；产品/池子维度可做**分类与筛选**（资产类型、期限、风险等级）。

### 1.4 Ondo Finance（RWA 产品与文档结构）

- **定位**：机构级、链上投资产品；多产品线（OUSG、USDY、Ondo Chain 等）。
- **产品与文档结构**：
  - **产品维度**：OUSG（短期美债代币化）、USDY（通用美元敞口）、Ondo Chain 等；每类产品有 Overview、Onboarding、KYC、Investing/Redeeming、Fees、Technical details。
  - **数据维度**：RWA.xyz 等第三方可展示 Ondo 的 TVL、holder、APY；官网与文档强调合规、托管、审计。
- **对我们的启示**：基金会平台「产品发布」可参考**按产品线 + 文档结构**（入门、KYC、投资/赎回、费用、技术）组织；官网侧可提供「产品与透明度」的索引，与第三方数据源（如 RWA.xyz 风格）衔接。

### 1.5 托管证明 / 储备证明（Proof of Reserve）类透明度

- **LedgerLens**：PoR 仪表盘，Merkle Tree 验证 + 第三方 attestation（如 The Network Firm LLP）；多资产（BTC、USDC、USDT、ETH）。
- **Ethena Transparency Dashboard**：Custodian attestations（Copper、CEFFU 等）；protocol backing ratio、system backing；Chainlink、Chaos Labs、LlamaRisk 等第三方验证。
- **Chainlink PoR Feeds**：三类数据源——第三方审计、托管方报告、自报 API（风险较高）；区分 offchain 与 cross-chain reserves。
- **对我们的启示**：MRD 中的「托管证明接口 (Proof of Custody)」可参考上述**第三方 attestation + 冷钱包/审计快照链接 + 标准化数据源**的呈现方式；官网与基金会平台池子详情页均可设「托管证明」入口。

## 二、Agent 相关 — API、MCP、开发者门户

### 2.1 Model Context Protocol (MCP) 在金融/Web3 的应用

- **协议角色**：开放、标准化的接口，让 AI Agent 与外部工具/API 无缝交互；常被类比为「AI 应用的 USB-C」。
- **Coinbase AgentKit + MCP**：AI Agent 可执行链上操作（token 转账、智能合约交互）；开发者通过 CDP API keys 在开发者门户认证；安装方式 npm、MCP SDK（@modelcontextprotocol/sdk）。
- **BNB Chain MCP**：专用 MCP 服务器，支持 blocks、transactions、wallet、on-chain actions；只读与可写工具、私钥认证。
- **RWA Pipe MCP**：Connect Claude Desktop 或任意 MCP client 到 **live RWA data**；配置示例 `claude_desktop_config.json` 中 `npx rwapipe-mcp`；无需 API key 即可基础接入，免费档有宽松限制。
- **对我们的启示**：官网「AI-Agent Kit & MCP 接入指南」需包含**MCP 是什么、我们提供哪些 MCP 能力（如 get_pool_apy、get_realtime_ltv、verify_kyc_status 等）、配置示例（Claude Desktop / 通用 MCP client）**；可参考 RWA Pipe 的「Multiple ways to integrate」+ 代码块 + 文档链接。

### 2.2 开发者门户与 API Key 管理（UI/UX 最佳实践）

- **自助创建 Key**：用户应能自行创建 API Key，而非仅通过工单申请。
- **组织级 Key**：支持组织级 API Key，与个人账号 Key 分离，便于团队共享且不暴露个人凭证。
- **Key 可见性**：两种设计——「仅展示一次、不可再查」（仅存 hash，更安全）与「可再次查看」（加密存储，更友好）；需在安全与体验间权衡。
- **必备能力**：Key 轮换（regenerate）、用量/活动监控（异常检测）、范围限制（域名白名单、产品/接口权限）、Key 删除、单应用单 Key（避免多系统共用一个 Key）。
- **安全**：仅存 hash、创建时仅展示一次 raw key、Key 前缀（如 pk_live / pk_test）便于识别；支持 OAuth 2.0 做委托接入。
- **对我们的启示**：官网「API Key 生成与管理、API 配置」控制台需支持创建/命名/权限/白名单/轮换/撤销/用量查看；若面向机构，需支持组织级 Key；文档中明确安全与范围限制说明。

### 2.3 AI Agent 在资金管理侧的呈现（老板思路对应）

- **趋势**：越来越多人通过 **AI Agent 打理资金**，Agent 会**自己找赚钱的项目**；平台需**对 Agent 友好**（可发现、可读懂、可推荐、可使用）。
- **RWA Pipe 首页话术**：「Built for developers, **AI agents**, and DeFi protocols」；「Connect Claude, GPT, or **AI agents** to live RWA data」。
- **对我们的启示**：官网与基金会平台均可在首屏或能力区明确「为 AI Agent 提供数据与接口」；基金会平台「用 AI Agent 连接」入口需可见、配置简单（如 MCP 配置示例、API 文档链接）；智能客服与「被 Agent 发现」并重。

## 三、成熟 UI/UX 技术栈与图标库（企业/金融场景）

### 3.1 企业级 Dashboard 技术栈（2024–2025 实践）

- **推荐核心**：**Next.js 14+（App Router）** + **React 19** + **TypeScript** + **Tailwind CSS** + **shadcn/ui**（基于 Radix UI）。
- **Next.js**：SSR、流式渲染、集成 API 路由、字体与图片优化；适合官网与需要 SEO/多页的基金会平台。
- **shadcn/ui**：组件代码可拷贝、无框架锁定、可深度定制；内置无障碍与深色模式；Vercel 背书，维护稳定。
- **数据与图表**：
  - **ECharts**：大数据量（10 万+ 点）表现好、渐进渲染与采样；MRD 已提，适合 LTV/NAV 等金融曲线。
  - **Recharts**：React 生态、易集成、tree-shaking 友好；适合中等数据量图表。
  - **Tremor**：基于 Tailwind 的仪表盘组件库，快速搭 KPI 与图表；可作补充。
- **表格**：**TanStack Table** 用于复杂排序、分页、筛选。
- **状态**：Redux Toolkit 或 Zustand 用于全局状态；服务端优先时可用 Server Components + 少量 client state。
- **实时**：WebSockets 或 Server-Sent Events 用于 LTV/NAV 等准实时更新。

### 3.2 图标库选型

- **Lucide**：1000+ SVG、React 官方支持、tree-shaking、无障碍；风格统一，适合企业/金融。
- **Heroicons**：企业场景常用，含货币、图表、交易等图标。
- **Phosphor Icons**：开源、多权重，可选。
- **React Icons**：聚合多库（Lucide、Feather、Material 等），按需引入。
- **建议**：主选 **Lucide** 或 **Heroicons**，与 shadcn/ui 风格一致；若有特殊金融图标可配合 React Icons 或自定义 SVG。

### 3.3 深色主题与设计系统（机构/金融）

- **MRD 要求**：保持 Prime 现有「机构、严谨、深色系/科技感」。
- **shadcn/ui**：内置 dark mode，配合 `next-themes` 做 light/dark/system 切换与持久化。
- **Ant Design**：企业级组件库，内置深色主题，可选作备选。
- **建议**：以 **shadcn/ui + Tailwind** 为主，定制一套 **深色为主** 的 token（背景、边框、文字、强调色），保证对比度与可读性；可选保留 light 模式切换。

### 3.4 多语言（EN + 繁体中文）

- **MRD 要求**：新增页面支持 **英文 (EN)** 和 **繁体中文 (TC)**。
- **Next.js**：Locale 路由（如 `/en/`、`/zh-TW/`）、middleware 做语言检测与重定向。
- **next-intl**：Next.js 常用 i18n 方案；namespace 化 JSON、`useTranslations`、多 locale（en、zh-TW）支持完善。
- **建议**：采用 **next-intl** 或 Next 官方 i18n 路由 + JSON 词典；统一日期、货币格式（如 Intl API）。
