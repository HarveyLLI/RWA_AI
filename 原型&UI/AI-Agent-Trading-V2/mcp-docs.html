1. MCP Server 安装与配置

Antalpha MCP Server 基于 Model Context Protocol 标准实现，为 AI Agent 提供标准化的理财操作接口。支持 stdio 和 HTTP 两种传输模式。

系统要求

  • Node.js >= 18.0
  • npm >= 9.0 或 yarn >= 1.22
  • 操作系统：macOS / Linux / Windows

快速安装

bash 
# 全局安装 MCP Server
npm install -g @antalpha/mcp-server

# 初始化配置
npx @antalpha/mcp-server init

# 启动服务（stdio 模式，适用于 Claude Desktop / Cursor 等）
npx @antalpha/mcp-server start --transport stdio

# 或 HTTP 模式（适用于自定义 Agent 集成）
npx @antalpha/mcp-server start --transport http --port 3000

配置文件

初始化后会在项目根目录生成 antalpha-mcp.config.json 配置文件：

json 
{
  "server": {
    "name": "antalpha-rwa",
    "version": "1.0.0",
    "transport": "stdio"
  },
  "auth": {
    "apiKey": "YOUR_API_KEY",
    "walletAddress": "0x..."
  },
  "protocol": {
    "endpoint": "https://api.antalpha.com/rwa/v1",
    "network": "ethereum",
    "defaultStrategy": "yield-btc-usdt"
  }
}

在 Claude Desktop 中配置

编辑 Claude Desktop 配置文件，添加 Antalpha MCP Server：

json 
{
  "mcpServers": {
    "antalpha-rwa": {
      "command": "npx",
      "args": ["@antalpha/mcp-server", "start"],
      "env": {
        "ANTALPHA_API_KEY": "your-api-key"
      }
    }
  }
}
提示：API Key 可在 Antalpha 平台「开发者设置」页面申请获取。每个 Key 绑定一个钱包地址。

2. MCP 协议详解

Model Context Protocol (MCP) 是一个开放标准协议，允许 AI 模型安全地与外部工具和数据源交互。Antalpha 基于 MCP 标准，为 AI Agent 提供以下能力：

核心概念

  • Tools（工具）：Agent 可调用的操作，如申购产品、查询持仓、发起赎回等
  • Resources（资源）：Agent 可读取的数据，如产品列表、收益率、链上地址信息等
  • Prompts（提示词模板）：预设的交互模板，帮助 Agent 高效决策

通信流程

流程 
┌──────────┐       MCP Protocol       ┌──────────────────┐
│ AI Agent │  ◄──── stdio / HTTP ────►  │ Antalpha MCP     │
│ (Claude) │                           │ Server           │
└──────────┘                           └────────┬─────────┘
                                                │
                                       ┌────────▼─────────┐
                                       │ Antalpha RWA API  │
                                       │ + Blockchain      │
                                       └──────────────────┘

3. Skills 应用方法

Skills 是对 MCP Tools 的高阶封装，提供预定义的投资策略与自动化工作流，让 Agent 无需关心底层 API 调用细节即可执行完整的投资逻辑。

Skill 与 Tool 的关系

  • Tool：原子操作，如「申购」「赎回」「查询」
  • Skill：由多个 Tool 组合的策略工作流，如「自动定投」「收益再投资」「到期自动赎回」

部署 Skill

bash 
# 查看可用 Skills 列表
npx @antalpha/mcp-server skills list

# 部署某个 Skill
npx @antalpha/mcp-server skills deploy --name yield-btc-auto

# 查看已部署 Skills 状态
npx @antalpha/mcp-server skills status

在 Agent 对话中使用 Skill

部署完成后，AI Agent 可以通过自然语言调用对应 Skill：

对话示例 
User: 帮我用 5000 USDT 申购 BTC-USDT 稳健型产品

Agent (via MCP):
  → 调用 get_product_detail("btc-usdt-stable")
  → 确认 APY: 6%, 锁定期: 6 个月
  → 调用 subscribe("btc-usdt-stable", 5000)
  → ✓ 申购成功，预计半年收益 150 USDT

自定义 Skill

你可以编写自定义 Skill，定义投资策略和触发条件：

javascript 
// skills/auto-invest.js
export default {
  name: "auto-invest",
  description: "每周自动定投 BTC-USDT 产品",
  triggers: [{ type: "cron", schedule: "0 9 * * MON" }],

  async execute(ctx) {
    const product = await ctx.call("get_product_detail", {
      product_id: "btc-usdt-stable"
    });

    if (product.apy >= 5.0) {
      await ctx.call("subscribe", {
        product_id: "btc-usdt-stable",
        amount: 1000
      });
      return { status: "invested", amount: 1000 };
    }
    return { status: "skipped", reason: "APY below threshold" };
  }
};
