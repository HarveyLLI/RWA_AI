# 项目三：管理平台搭建要点与合约对接说明

> 目的：项目三作为**管理后台**，配合项目二（用户端产品站）的运营，负责产品信息发布、配置与监控；合约团队独立开发智能合约。本文档说明管理平台需搭建的关键点，以及**项目二前端**、**项目三后端**分别在哪些环节与合约对接。

---

## 一、项目二当前能力回顾（管理平台要支撑什么）

项目二用户端主要能力包括：

| 能力 | 涉及数据/配置 | 数据来源倾向 |
|------|----------------|--------------|
| 产品列表（markets） | 金库列表、到期日、APY、TVL、$ANTA 折扣、状态 | 后端 API + 链上/预言机 |
| 产品详情（vault-detail） | 单库详情：NAV、抵押、风控参数、合约地址、审计、最近交易 | 后端 API + 链上 |
| 用户存入 | 金额、金库、钱包签名 | **合约** |
| 用户持仓 / 仪表盘 | 用户各库持仓、净值、到期日、可赎回 | 链上读 + 后端聚合 |
| 到期赎回 / 兑换 $ANTA | 用户操作、钱包签名 | **合约** |
| Connect AI / API Key | Skills 数据、Trading API、Key 管理 | 后端 API |
| 净值走势、统计指标 | NAV 历史、30 日回报、成立以来表现 | 后端或预言机 |

因此：**管理平台需要对这些信息的“发布、配置、监控、审计”负责**，并在合适环节把链上不可变能力交给合约，把可配置与可展示数据交给后端与前端。

## 二、项目三（管理平台）需搭建的关键点

### 2.1 产品 / 金库管理（核心）

| 关键点 | 说明 | 与项目二的关系 |
|--------|------|----------------|
| **金库的创建与上下架** | 新增金库（绑定合约地址、代币符号、到期日等）、上架/下架/排序 | 项目二产品列表与详情的数据源 |
| **金库基础信息维护** | 名称（中/英）、代币符号（如 AA-00330、wlUSDT-Jun26）、到期日、网络（Ethereum 等）、展示状态 | 列表页、详情页、仪表盘展示 |
| **收益与折扣配置** | 到期 APY、$ANTA 兑换折扣率（或从合约/策略读取后在此覆盖展示） | 项目二多处展示 APY、折扣 |
| **合约与链上标识** | 金库合约地址、收益凭证代币地址、DEX 池地址（若有） | 前端调合约、Etherscan 链接、API 返回 |
| **风控与披露参数** | 资金需求方（Originator）、抵押覆盖率、LTV、清算阈值、托管方、预言机、审计信息、链上证明哈希 | 详情页「配置与风控」「底层抵押」 |
| **NAV 与业绩** | 当前净值（NAV）、30 日回报、成立以来回报、预计年化（或由后端按策略计算） | 详情页指标与净值走势图 |

**建议**：  
- 链上不可变或由预言机/合约维护的数据（如实时 TVL、链上 NAV）：管理端以**只读展示 + 告警**为主。  
- 展示用文案、排序、上下架、审计报告链接、证明哈希等：管理端**可编辑**，通过项目三后端 API 提供给项目二。

---

### 2.2 内容与运营配置

| 关键点 | 说明 | 与项目二的关系 |
|--------|------|----------------|
| **首页 / 全局配置** | 协议总锁仓、在售产品数、APY 区间等统计文案或数字（可来自链上汇总或人工配置） | 项目二首页 Hero、统计区块 |
| **公告与风险提示** | 存/赎/换 $ANTA 的说明、免责文案、合规话术 | 弹窗、帮助中心、页脚 |
| **帮助 / FAQ** | 参与流程、退出方式、API 与 AI Agent 说明 | 项目二帮助页、AI 回复话术可考虑从后端拉 |

---

### 2.3 交易与链上数据监控（只读 + 告警）

| 关键点 | 说明 | 与项目二的关系 |
|--------|------|----------------|
| **交易流水 / 活动** | 存入、赎回、兑换 $ANTA 等事件（来自链上事件或后端索引） | 详情页「最近交易」、仪表盘可扩展 |
| **TVL / 余额监控** | 各金库 TVL、用户持仓汇总（读合约或索引） | 列表与详情的 TVL、用户仪表盘 |
| **NAV 与预言机** | 若 NAV 由预言机或合约更新，管理端展示当前值及历史，异常告警 | 详情页净值、净值走势 |
| **风控与清算告警** | LTV、抵押覆盖率、清算线接近时告警 | 运营与风控使用，不直接上用户端 |

**建议**：  
- 项目三后端**订阅或轮询合约事件**，建索引库，为项目二提供「最近交易」等 API。  
- 管理端大屏或列表展示各库 TVL/NAV/告警状态，便于运营与合约/风控协作。

### 2.4 用户与权限（若需要）

| 关键点 | 说明 | 与项目二的关系 |
|--------|------|----------------|
| **管理员账号与权限** | 登录、角色（如运营/风控/超管）、操作审计日志 | 仅管理平台使用 |
| **API Key 管理（可选）** | 若 AI API Key 由后端签发与校验，管理端可查看 Key 列表、绑定钱包、撤销 | 与项目二「仪表盘 — AI API Key」对应；若 Key 纯前端/合约则可不做 |

---

### 2.5 Connect AI 与开放数据

| 关键点 | 说明 | 与项目二的关系 |
|--------|------|----------------|
| **Skills 数据源** | get_vault_list、get_yield_curve、get_vault_apy、get_spot_price、get_risk_metrics、get_custody_proof 等接口的数据来源 | 项目三后端（或独立服务）提供 API，项目二/第三方 Agent 调用 |
| **Trading API 与鉴权** | 存/赎/换等交易接口的 API Key 校验、限流、日志 | 项目三后端鉴权，交易执行仍由前端调合约 |

---

### 2.6 小结：管理平台关键模块清单

1. **金库/产品管理**：创建、编辑、上下架、排序；基础信息、收益与折扣、合约与风控参数、NAV/业绩展示配置。  
2. **内容与运营**：首页与全局统计、公告、风险提示、FAQ/帮助。  
3. **链上与交易监控**：交易流水索引、TVL/NAV 监控、风控与清算告警（只读 + 告警）。  
4. **管理员与权限**：登录、角色、操作审计；可选 API Key 管理。  
5. **Connect AI 数据与鉴权**：Skills 类 API 数据、Trading API 鉴权与日志。

## 三、合约团队提供的合约能力（假设）

以下按常见 RWA/收益产品设计假设，实际以合约团队交付为准：

| 合约/能力 | 说明 |
|-----------|------|
| **金库合约（按到期日）** | 每只产品一个金库：存入 USDT 铸造收益凭证（如 wlUSDT-Jun26）、到期赎回凭证兑 USDT、或兑换 $ANTA。 |
| **ERC-20 收益凭证** | 每个金库对应一种凭证代币，可查询余额、总量（TVL 相关）。 |
| **NAV / 预言机** | 当前净值由预言机或合约内部逻辑更新，可被读。 |
| **$ANTA 兑换** | 凭证按规则销毁并铸造/转出 $ANTA（可能单独合约或集成在金库）。 |
| **风控/抵押** | 底层抵押（如 BTC）的 LTV、清算阈值等可能由预言机或风控合约提供只读数据。 |

## 四、项目二前端 — 在哪些环节连接/调用合约

项目二前端（用户端）**直接与链交互**的环节如下：

| 环节 | 前端动作 | 合约/链上调用 | 说明 |
|------|----------|----------------|----------------|
| **连接钱包** | 请求连接（如 MetaMask / WalletConnect） | 仅获取地址，不调合约 | 地址用于展示、后端鉴权、链上读持仓。 |
| **产品列表 / 详情展示** | 拉取金库列表、单库详情、NAV/TVL 等 | **可选**：前端直接读合约（TVL、NAV、总供应量）作为补充或降级 | 建议以**项目三后端 API**为主，前端可选择性读链上 TVL/NAV 做实时性增强。 |
| **存入 USDT** | 用户输入金额 → 弹框（风险核查等）→ 发起交易 | 1. **Approve**：用户对 USDT 授权给金库合约<br>2. **Deposit**：调用金库合约的 deposit(uint256) 或等价方法 | 合约团队需提供 ABI、方法名、金库合约地址；前端用 ethers/viem 等发起交易。 |
| **查看持仓** | 仪表盘 / 详情页「我的持仓」 | **读合约**：查询用户在某金库凭证代币上的 **balanceOf(userAddress)**，结合 NAV 算当前价值 | 凭证合约地址、ABI；NAV 可来自后端 API 或链上读。 |
| **到期赎回** | 用户点击「到期赎回」 | **写合约**：调用金库合约的 **redeem** 或类似方法，销毁凭证、转回 USDT | 需到期后可用；前端需读合约判断是否已到期。 |
| **兑换 $ANTA** | 用户确认风险后发起兑换 | **写合约**：调用兑换合约或金库的 **swapToAnta** 等，销毁凭证、铸造/转出 $ANTA | 合约团队提供接口与前端集成。 |
| **最近交易 / 活动** | 展示最近存入、赎回、兑换记录 | **可不直接调合约**：由项目三后端索引链上事件后通过 API 提供给前端 | 若前端要自给自足，可读合约 **Transfer / Deposit / Redeem** 等事件。 |

**汇总**：  
- **必须与合约对接的前端环节**：存入（Approve + Deposit）、到期赎回（Redeem）、兑换 $ANTA（Swap）。  
- **建议前端调合约只读**：用户持仓（balanceOf）、可选 TVL/NAV（若后端未提供或需实时）。  
- **建议从前端剥离到后端**：金库列表与详情、NAV 历史、最近交易、风控参数等由项目三后端统一提供，前端只调 API。

## 五、项目三后端 — 在哪些环节连接/调用合约

项目三后端主要做**数据聚合、索引、配置下发与鉴权**，与合约的衔接如下：

| 环节 | 后端动作 | 合约/链上调用 | 说明 |
|------|----------|----------------|----------------|
| **金库配置与地址** | 管理端配置金库合约地址、凭证代币地址、网络 | 仅存库，不直接调合约 | 配置结果给前端与后端自身使用。 |
| **产品列表 / 详情 API** | 返回金库列表、单库详情（名称、APY、折扣、风控文案等） | **可选**：从链上读各金库 TVL、当前 NAV，与库内配置合并后返回 | 建议后端定时或按需读链上 TVL/NAV，保证项目二展示一致。 |
| **NAV 与净值走势** | 提供当前 NAV、历史 NAV（净值走势图） | **读合约或预言机**：按金库地址读当前 NAV；历史 NAV 若链上可查则索引，否则由运营/预言机写入后端 | 合约或预言机需提供 currentNAV()、或事件/历史数据源。 |
| **交易流水 / 最近交易** | 提供「最近存入/赎回/兑换」列表 | **索引链上事件**：订阅或轮询金库合约的 Deposit、Redeem、Swap 等事件，写入 DB，再通过 API 给项目二 | 合约团队需明确事件名、参数（用户、金额、时间等）。 |
| **用户持仓聚合（可选）** | 为「仪表盘」提供用户在各金库的持仓、净值、价值 | **读合约**：对每个金库凭证合约调用 balanceOf(userAddress)；NAV 由后端提供 | 后端需有用户地址（前端登录/连接钱包后上报或 API 带地址查询）。 |
| **TVL / 统计** | 首页与列表的 TVL、协议总锁仓 | **读合约**：各金库总供应量或总锁仓视图，汇总 | 合约需提供 totalSupply() 或等价只读接口。 |
| **风控与清算** | 展示 LTV、抵押覆盖率、清算线 | **读预言机/风控合约**：若数据在链上，后端定时拉取并告警 | 合约团队提供预言机或风控只读接口。 |
| **写入/管理类（若有）** | 如运营角色更新 NAV、触发迁移、暂停 | **写合约**：仅当合约设计上有 admin 方法且由后端托管私钥或多签时 | 需与合约团队明确权限模型，通常不推荐后端直接持私钥。 |

**汇总**：  
- **后端主要用合约做只读**：TVL、NAV、用户持仓（balanceOf）、事件索引（Deposit/Redeem/Swap）。  
- **后端负责**：金库与内容配置、事件索引、API 鉴权、NAV 历史与走势数据、告警。  
- **后端与合约的衔接点**：金库/凭证合约地址、ABI、事件定义、预言机或风控只读接口。

## 七、建议的落地顺序（项目三 + 合约对接）

1. **合约团队**：明确并交付金库合约（deposit/redeem）、凭证代币、兑换 $ANTA 接口及 ABI、事件定义；若 NAV/风控在链上，提供只读接口或预言机地址。  
2. **项目三后端**：  
   - 先做金库/产品管理（含合约地址配置）与列表/详情 API，供项目二前端对接；  
   - 再做链上事件索引与「最近交易」API、TVL/NAV 只读聚合（若合约已就绪）；  
   - 最后做 Connect AI 数据与鉴权、管理端权限与审计。  
3. **项目二前端**：  
   - 产品列表与详情先接项目三 API（可 mock）；  
   - 合约就绪后接入：存入、赎回、兑换 $ANTA 的写操作，以及持仓、可选 TVL/NAV 的读操作。
