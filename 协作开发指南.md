# RWA&AI 项目 — 与朋友携手共建指南

本文说明如何把本仓库推到远程并邀请协作者一起维护。

---

## 一、你（项目所有者）需要做的事

### 1. 在项目目录下初始化 Git 并首次提交

在终端执行（注意：路径里的 `&` 在终端中要加引号或转义）：

```bash
cd "/Users/harvey/Documents/Cursor/RWA&AI"
git init
git add .
git commit -m "初始提交：RWA&AI 项目文档与结构"
```

### 2. 创建远程仓库并推送

任选一个平台创建**新仓库**（不要用 README 初始化）：

| 平台   | 网址                 | 说明           |
|--------|----------------------|----------------|
| Gitee  | https://gitee.com    | 国内访问快     |
| GitHub | https://github.com   | 国际常用       |
| GitCode| https://gitcode.com  | 国内免费       |

- 仓库名可填：`RWA-AI` 或 `rwa-ai-project` 等（部分平台仓库名不建议用 `&`）。
- 创建后复制仓库的 **HTTPS 地址**，例如：`https://gitee.com/你的用户名/RWA-AI.git`

在本地执行（把下面地址换成你的）：

```bash
cd "/Users/harvey/Documents/Cursor/RWA&AI"
git remote add origin https://gitee.com/你的用户名/仓库名.git
git branch -M main
git push -u origin main
```

### 3. 邀请朋友为协作者

- **Gitee**：仓库 → 管理 → 仓库成员管理 → 添加成员，权限选「开发者」或「主开发者」。
- **GitHub**：仓库 → Settings → Collaborators → Add people，输入对方 GitHub 用户名或邮箱。

对方接受邀请后即可克隆、修改并推送代码。

---

## 二、你的朋友需要做的事

### 1. 克隆项目到本地

```bash
git clone https://gitee.com/你的用户名/仓库名.git
cd 仓库名
```

### 2. 日常协作流程

```bash
# 开始改之前先拉取最新
git pull

# 改完后提交并推送
git add .
git commit -m "简短描述本次修改"
git push
```

---

## 三、推荐协作习惯

1. **改前先 `git pull`**，减少冲突。
2. **小步提交**：完成一块内容就 commit 一次，备注写清楚。
3. **按目录分工**：例如一人负责「需求文档 / 业务资料」，另一人负责「原型&UI」或「执行代码」，减少同一文件同时被改。
4. **遇冲突**：打开冲突文件，按 Git 提示保留/合并内容，再 `git add`、`git commit`。

---

## 四、可选：用分支开发

- 保持 `main` 为稳定可交付版本。
- 新功能开分支：`git checkout -b feature/功能名`，开发完成后再合并回 `main` 并推送。

---

按以上步骤完成后，你和朋友即可基于同一份 RWA&AI 仓库协同编辑文档与代码。有问题可在仓库的 Issues 中记录。
