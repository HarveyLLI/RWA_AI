# 三种 CeFi 产品打包为债与线上募资逻辑

> 整理日期：2026-02-26  
> 来源：您补充；与昨日流程图、借贷与 RWA 逻辑对齐。  
> 用途：明确「三种产品 → 债项目 → 线上募资 → 投资者收益 + 平台串联 + AI Agent」的完整理解，供产品与汇报使用。

---

## 一、核心理解（您要表达的逻辑）

- **未来产品形态**：把我们现有的三种 **CeFi 产品**（**质押物贷款、电费贷款、矿机贷款**）**打包成债项目**，在**未来搭建的网站**（基金会平台）上**募资**；**投资客户可以在线购买这些产品**，**获得收益**。
- **平台的角色**：把**整条链条串通起来**（借贷端 → 债的生成与审核 → 募资端 → 投资者购买与收益），并**支持 AI Agent 捕捉到我们这样的高收益理财渠道，与我们交易**。

---

## 二、举例说明（矿机贷款 → 债 → 线上募资）

| 环节 | 说明 |
|------|------|
| **借款端** | **借款人 A** 找我们公司借 **100 万**，作为他**购买矿机的启动资金**；我们与 A 约定**年化 10%**。 |
| **债的生成** | 我们将这笔借贷（或同类多笔）打包成一个**债项目**，通过合约标准化、代币化，变成可在链上/线上销售的**理财产品**。 |
| **上线募资** | 把这个产品发到**未来搭建的网站**（基金会运营的链上募资平台）上**募资**。 |
| **投资端** | **投资客户**在线上**投资/购买**该产品，可能获得例如 **7%** 的收益（收益来自借款人 A 的还本付息）。 |
| **利差与链条** | 借款端 10% 与投资端 7% 之间的利差，体现我们与基金会的服务与风控价值；募资所得作为**放款资金**给到借款人 A，整条链打通。 |

**平台的价值**：从借款人申请 → 我们尽调与风控 → 债项目生成与审核 → 基金会平台上线募资 → 投资者购买与持有 → 还本付息分配，**全链条在一个体系内串通**；同时**对 AI Agent 开放**，让 Agent 能**发现这类高收益渠道并与我们交易**（即 Agent 代表投资者发现产品、下单、获得收益）。

---

## 三、与昨日流程图的一一对应

| 流程图中的角色/步骤 | 在本例中的对应 |
|----------------------|----------------|
| **借款人** | 借款人 A（借 100 万购矿机，年化 10%）。 |
| **债权工厂（Originator / NS）** | 我们公司：产生这笔矿机贷款订单，持有/管理抵押品（或对应权益），与基金会签 MLA。 |
| **质押抵押品 / 准入与尽调** | 矿机或相关抵押/担保；我们做准入与尽调，信息提交 AA 平台。 |
| **AA 平台（技术提供方 + 合约开发方）** | 我们提供的技术：尽调与风控、审核(ASA)、债项目/资产池构建（智能合约）、分配资金(MLA) 给离岸主体。 |
| **离岸运营主体（基金会）** | 链上募资平台的运营方；接收 MLA 分配的资金，在**网站上**向投资客户**募资**（卖债/LP token/股票 Token）。 |
| **Web3 用户 / 机构** | **投资客户**：在线上购买我们的债产品，获得约 7% 收益等。 |
| **AI Agent** | **捕捉我们这样的高收益理财渠道**，发现产品、与我们（或基金会平台）**交易**，代表用户投资。 |

三种 CeFi 产品（质押物贷款、电费贷款、矿机贷款）均可按同一逻辑：**单笔或打包成债项目 → 上架基金会网站募资 → 投资客户线上购买获收益 → 平台串联全链 + 支持 AI Agent 发现与交易**。

---

## 四、对产品与文案的归纳

- **债项目来源**：来自我们的**三种 CeFi 产品**（质押物贷款、电费贷款、矿机贷款），可单笔或打包成池。
- **投资客户**：在**未来搭建的网站**（基金会平台）上**在线购买**这些债产品，**获得收益**（如例中的 7%）。
- **平台**：**串联整条链**（借贷 → 债 → 募资 → 投资 → 收益分配），并**支持 AI Agent 发现高收益渠道并和我们交易**。
- **官网与基金会平台**：官网展示技术能力与透明度（债从哪来、风控如何）；基金会平台是**投资客户真正下单购买**的场所；AI Agent 通过 API/数据发现产品，在合规与授权下与我们或平台交易。
