# 产品定位与亮点方向

> 整理日期：2026-02-26  
> 用途：明确「沿老板思路推进 + 增加产品亮点、提升主营业务吸引力」的定位，并提炼可落地的亮点方向，供设计与呈现参考。

---

## 一、产品定位（您的要求）

- **主基调**：**沿着老板的思路来做**——RWA/债/合约、技术赋能与风控预言机、双站（官网 + 基金会平台）、AI 参与（推广、被 Agent 发现、智能客服）等，按既定方向推进。
- **在推进中的增强**：**多一些产品亮点**，为**主营业务**（借贷 → 债 → 募资、风控、透明度）**添加更多吸引力**，让资金方、投资者、合作方更愿意选择我们。
- **与反思的关系**：业务发展反思（需求验证、资金方多元化、分阶段等）可作为**执行时的补充思路**，不改变「沿老板思路」的主线；亮点设计则让这条主线**更有说服力、更易获客**。

---

## 二、主营业务吸引力可以落在哪些地方？

主营业务 = **借贷（资产端）→ 债的转化 → 募资（资金端）** + **风控与透明度** + **技术能力与合规**。吸引力可以从「资金方/投资者/合作方最关心什么」反推，在**同一套业务**上做出**更清晰、更可信、更易用**的呈现。

---

## 三、产品亮点方向（供设计时选用）

以下为在**不偏离老板思路**的前提下，可加强的**产品亮点**，用于提升主营业务吸引力；设计官网、基金会平台、AI、披露与话术时可选用或组合。

### 1. 透明度与可信度（对应「风控、资产真实」）

| 亮点方向 | 说明 | 可落地处 |
|----------|------|----------|
| **一目了然的全局指标** | TVL、BTC 抵押量、Global LTV、清算线等，**实时或准实时**、大屏或首屏展示，让「资产真实、超额抵押」一眼可见。 | 官网新导航下 RWA 透明度看板、基金会平台首页 |
| **资金池透视** | 每池独立详情：Originator KYC/评级、托管证明（冷钱包/审计快照）、链上 NAV/APY，**结构化、可追溯**。 | 官网透明度看板、基金会平台池子详情页 |
| **对 Agent 友好** | 结构化数据、API/文档、清晰字段，让「找收益」的 Agent 能发现、读懂、推荐我们 → 对资金方而言也是「可被第三方验证」的信任信号。 | 官网开发者/AI 门户、基金会平台产品与风险说明 |

### 2. 合规与身份（对应「合规、KYC/DID」）

| 亮点方向 | 说明 | 可落地处 |
|----------|------|----------|
| **链上 DID / KYC 展示** | 明确展示我们支持的链上 DID、KYC 工具包，以及「零知识证明保护隐私、满足监管」的说明，增强机构与合规敏感型资金方的信心。 | 官网合规与 DID 工具箱、披露文案 |
| **合规话术统一** | 全站统一用 Infrastructure / Monitor / Verify / Technology，避免 Invest / Deposit / Earn；页脚 RWA 免责清晰 → 形成**专业、合规**的品牌印象。 | 官网、基金会平台、所有对外文案 |

### 3. 体验与效率（对应「客户沟通、智能客服、AI」）

| 亮点方向 | 说明 | 可落地处 |
|----------|------|----------|
| **智能客服与自助** | 借款端：产品与流程说明、自助查询（抵押率、建议操作）、申请引导；投资者端：常见问题、持仓与验证资产安全 → **7×24、响应快**，降低沟通成本。 | 官网、基金会平台（设计/开发必含项） |
| **AI 推广与内容** | 用 AI 生成推广文案、产品与能力介绍、多语言素材，**统一、专业、合规**，让官网与平台呈现更一致、更易传播。 | 官网新导航、基金会平台落地页、社媒/投教 |
| **「用 AI Agent 连接」** | 明确入口与说明：投资者可用自己的 Agent 监控、告警、在授权下操作；我们提供 API/数据与风控能力 → **差异化卖点**，吸引「用 Agent 打理资金」的客群。 | 基金会平台、官网能力介绍 |

### 4. 生态与信任（对应「合作方、Powered by」）

| 亮点方向 | 说明 | 可落地处 |
|----------|------|----------|
| **Powered by 生态墙** | 展示合作 DApp/基金会、合作方 logo 与简短说明，**合规离站弹窗**（第三方运营、我们仅技术+风控），既展示生态又控制合规风险。 | 官网主页/新导航下 |
| **合作方与背书** | 战略合作、审计方、托管方（如 Cobo/Anchorage）、DID 供应商等，**有图有据**，增强「机构级」印象。 | 官网技术能力、合规与 DID、透明度页 |

### 5. 产品与结构（对应「债、池子、收益」）

| 亮点方向 | 说明 | 可落地处 |
|----------|------|----------|
| **池子与收益清晰** | 期限、APY、风险等级、抵押品类型、托管证明入口，**结构化展示**，便于资金方/投资者快速比较与决策。 | 基金会平台产品列表与详情 |
| **分层与选择** | 若有优先/劣后或不同风险档位，**清晰标注**，吸引不同风险偏好的资金方，扩大可触达资金池。 | 基金会平台、官网产品说明 |

---

## 四、小结：沿老板思路 + 亮点怎么用

- **老板思路**：RWA/债/合约、技术赋能与风控预言机、双站、AI 参与、合规与 DID 等 → **主线不变**。
- **产品亮点**：在**同一套主营业务**上，把**透明度、合规、体验、生态、产品结构**做得更清晰、更可信、更易用，形成「机构级、可验证、对 Agent 友好、服务到位」的**整体吸引力**。
- **执行时**：做官网新导航、基金会平台、AI、披露与文案时，从上述亮点方向中**选用或组合**，不必一次全上，优先选**资金方/投资者最在意**的（如透明度、合规、智能客服、「用 Agent 连接」），再逐步加生态、分层等。
