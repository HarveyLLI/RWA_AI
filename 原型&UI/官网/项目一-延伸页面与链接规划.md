# 项目一：延伸页面与链接规划

> 整理日期：2026-02-26  
> 用途：说明当前介绍页（Landing）需要哪些额外页面与链接来支撑完整性，便于后续产品与开发对齐。

---

## 一、当前页面的跳转与支撑需求

### 1. Explore RWA Tech（Hero 主按钮）

- **当前行为**：锚点跳转到本页「How It Works」区块（`#how-it-works`）。
- **建议后续**：
  - **方案 A**：跳转到「RWA Tech 概览」独立页，例如 `/web3-infrastructure/overview` 或 `/web3-infrastructure`，用一页把运作方式、技术能力、透明度、开发者入口串起来。
  - **方案 B**：保持锚点至「How It Works」，若整站只有当前 Landing 单页，则无需改。
- **落地**：在 HTML 中已预留 `data-dest="PLAN: /web3-infrastructure/overview"`，上线前将 `href` 改为实际路由或 URL。

---

### 2. Earn products on the platform（透明度看板下的产品列表）

- **当前行为**：标题与四张产品卡片均为占位锚点 `#foundation-products`，无真实跳转。
- **建议后续**：
  - **标题「Earn products on the platform」**：链接到**基金会产品官网**的「产品列表」或首页（由业务确定具体基金会域名与路径）。
  - **每张产品卡片「View details」**：链接到该基金会官网下**对应产品的详情页**（如 BTC-Backed Earn、Miner Financing Pool、Power Cost Financing、Multi-Asset Earn 各自一条链接）。
- **支撑页面**：需要**基金会侧**已有（或规划）产品官网，提供「产品列表 + 单产品详情」页面；本页只做入口与数据展示，不替代基金会官网。
- **落地**：HTML 中标题与卡片已使用 `href="#foundation-products"` 与 `data-dest="PLAN: 基金会产品官网"`，后续替换为基金会产品站真实 URL（可统一一个 base URL，产品详情用 path 或 query 区分）。

---

### 3. 合作方卡片（Powered by Antalpha Prime 下的 BITMAIN、NORTH STAR、BitFuFu 等）

- **当前行为**：点击任意合作方卡片 → 弹出「离站提示」合规弹窗（文案含该合作方名称）→ 用户确认后，若已配置 `data-partner-url` 则跳转，否则仅关闭弹窗并提示“生产环境将跳转”。
- **建议后续**：
  - 每个合作方卡片应跳转到**该合作方 / 基金会的官方网站**（由商务或运营提供正式 URL）。
  - 在 HTML 中为每个 `.partner-card` 的 `data-partner-url` 填入对应官网地址；`data-partner-name` 已用于弹窗文案，无需改。
- **支撑**：无需新建页面，仅需配置 9 个合作方的官方链接；合规弹窗已实现，可复用。
- **落地**：在 `index.html` 中为各卡片补充 `data-partner-url="https://..."` 即可；暂无 URL 的可先留空，行为与现在一致。

---

### 4. Smart Contract Hub 与 AI-Agent Kit & MCP（开发者门户）

- **当前行为**：「View audit reports」「MCP documentation」以及底部「Open API & MCP docs」均为占位链接（`#docs`），无真实文档站。
- **建议后续**：
  - **需要单独建设「开发者文档站」**（可子路径如 `/web3-infrastructure/docs` 或独立域名如 `docs.xxx.com`），至少包含：
    - **Smart Contract Hub**：智能合约架构说明、审计报告、GitHub 链接、部署与调用说明。
    - **AI-Agent Kit & MCP**：  
      - 公共 API 列表与说明；  
      - API 调用方式（鉴权、请求示例、错误码等）；  
      - MCP（Model Context Protocol）接入指南、接口清单、示例代码与快速启动。
  - 本页「Developer & AI-Agent Portal」区块作为入口，链接到该文档站的首页或对应章节；「View audit reports」可链到审计报告页，「MCP documentation」可链到 MCP/API 章节。
- **支撑页面**：**文档站**（含公共 API 说明、API 调用方法、MCP 使用说明），与介绍页并列，共同支撑「技术能力 + 开发者/AI 接入」的完整性。
- **落地**：HTML 中已加说明文案「Full API reference and MCP usage are available in our documentation site」及占位链接与 `data-dest`；后续将 `href` 改为文档站实际 URL。

---

### 5. Compliance & DID Tools

- **结论**：已从当前介绍页**移除**该模块（原 KYC/AML、DID、ZKP 展示区块），不再在 Landing 上展示；若后续合规/品牌需要，可放在文档站或单独合规说明页，不作为本页必需支撑。

---

## 二、延伸页面与站点汇总

| 类型           | 说明与建议链接目标 |
|----------------|--------------------|
| **RWA Tech 概览页** | 可选。若「Explore RWA Tech」改为独立页，则需该页；否则保持锚点即可。 |
| **基金会产品官网** | 必须。Earn 产品列表标题 + 每张产品卡片的「View details」链到该站的产品列表与单产品详情。 |

| **Compliance & DID** | 已从本页去掉；如需可单独做合规页或放入文档站。 |
