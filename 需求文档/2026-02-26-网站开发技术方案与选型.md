# 网站开发技术方案与选型

> 整理日期：2026-02-26  
> 用途：为老板汇报用网站 demo 及后续生产环境提供成熟、稳定的 UI/UE 技术栈与图标库等选型；从专业角度出发，优先稳定与体验，不因节省费用牺牲质量。  
> 依据：竞品调研（RWA 与 Agent 相关）、MRD 非功能性需求、项目目标与交付标准。

---

## 一、设计原则与约束（您的要求）

- **受众**：网站是**给老板看的**，需呈现**成熟、专业**的 UI/UE。
- **技术选型**：采用**成熟的 UI/UE 技术栈**，提前考虑好框架、组件库、图表、图标等。
- **专业与稳定**：从**专业角度**出发；**您授权**可自主调用 skill、采用**先进且稳定**的技术；**稳定性优先**。
- **模型与成本**：在自动模式下采用模型时，**使用合适的模型**，**不要从节省费用的角度出发**——优先质量与效果。
- **合规与品牌**：保持 Prime 现有「机构、严谨、深色系/科技感」品牌调性；数据图表需高性能、大数据量不卡顿；多语言 EN + 繁体中文。

## 二、推荐技术栈总览

| 层级 | 选型 | 说明 |
|------|------|------|
| **框架** | Next.js 14+ (App Router) | SSR/流式/API 路由/优化；适合官网与多页应用，SEO 友好。 |
| **语言** | TypeScript | 端到端类型安全，便于维护与交接开发。 |
| **UI 基础** | React 19 | 当前稳定版，并发与渲染优化。 |
| **样式** | Tailwind CSS | 原子化、一致、易定制；与 shadcn/ui 配套。 |
| **组件库** | shadcn/ui（Radix UI 基座） | 可拷贝代码、无锁定、可深度定制；无障碍与深色模式内置；Vercel 背书。 |
| **图表** | ECharts（主）/ Recharts（辅） | MRD 要求高性能图表；ECharts 适合 LTV/NAV 等大数据量曲线；Recharts 适合 React 内嵌中小图表。 |
| **表格** | TanStack Table | 复杂排序、分页、筛选，企业级表格标配。 |
| **图标** | Lucide（主）/ Heroicons（辅） | 风格统一、tree-shaking、无障碍；金融/企业场景常用。 |
| **深色主题** | next-themes + shadcn/ui 深色 token | 机构感、科技感；支持 light/dark/system 与持久化。 |
| **多语言** | next-intl | EN + 繁体中文 (zh-TW)；路由与文案分离，与 Next.js App Router 适配。 |
| **状态** | Zustand 或 Redux Toolkit | 全局状态；服务端优先处用 Server Components。 |
| **实时数据** | WebSockets 或 Server-Sent Events | LTV/NAV 等准实时更新（可选，按需求接入）。 |

## 三、分项说明与备选

### 3.1 框架与运行时

- **Next.js 14+ (App Router)**：服务端渲染、流式、中间件、集成 API；适合官网（项目一）与基金会平台（项目二）的多页、多路由；社区与生态成熟，便于招人维护。
- **备选**：若项目二（基金会平台）需强交互与 SPA 体验，可仍用 Next.js 做前端，或单独用 Vite + React 构建；建议**先统一 Next.js**，降低双栈维护成本。

### 3.2 组件与设计系统

- **shadcn/ui**：组件以源码形式拷贝到项目中，非 npm 黑盒依赖；基于 Radix UI，无障碍与键盘导航完善；**深色模式**通过 CSS 变量与 `next-themes` 实现；与 Tailwind 深度集成。
- **设计 token**：在 Tailwind 配置中定制**深色为主**的 palette（背景、边框、文字、强调色、图表色），保证对比度与「机构、科技感」；与 MRD「深色系/科技感」一致。
- **备选**：Ant Design 适合强表单与后台；若团队已有 Ant Design 沉淀可考虑，否则 shadcn/ui 更轻、更易定制品牌。

### 3.3 图表库

- **ECharts**：MRD 明确提及；适合**大数据量**（如 LTV 曲线、NAV 历史）；支持 canvas/SVG、渐进渲染与采样；需在 React 中封装或使用 echarts-for-react 等封装库。
- **Recharts**：React 原生、声明式、易集成；适合**中小数据量** KPI 与折线/柱状图；可与 ECharts 分工：复杂/大数据用 ECharts，简单指标用 Recharts。
- **Tremor**：基于 Tailwind 的仪表盘组件，可快速做 KPI 卡片与简单图表；可作为补充，非必选。

### 3.4 图标库

- **Lucide**：1000+ 图标、React 官方支持、tree-shaking、风格统一；适合导航、按钮、状态、金融相关图标。
- **Heroicons**：企业/金融场景常用；若需与现有 Prime 风格一致可对照选用。
- **使用方式**：按需按名导入（如 `import { ArrowRight, Shield } from 'lucide-react'`），避免全量打包。
- **备选**：React Icons 聚合多库；若需特殊金融图标可配合使用；Iconify 可统一多套图标集，按需选用。

### 3.5 深色主题与多语言

- **深色主题**：使用 `next-themes` 的 ThemeProvider；shadcn/ui 的 `dark` 类与 CSS 变量切换；默认以**深色**为主，提供 light/system 切换（若产品需要）。
- **多语言**：采用 **next-intl**；路由形式 `/en/...`、`/zh-TW/...`；文案按 namespace 组织（如 `common.json`、`dashboard.json`）；日期、货币、数字用 `Intl` 或 next-intl 的 formatters，与 EN/zh-TW 一致。

## 四、项目结构建议（执行代码目录）

- **单仓多应用**（推荐）：根目录一个 Next.js 工程，通过路由或 path 区分「官网新导航下内容」与「基金会平台」；共享组件、样式、工具。
- **或双仓**：`执行代码/antalpha-web`（官网）、`执行代码/foundation-platform`（基金会平台）；共享 UI 与工具可抽成 npm 包或 monorepo 子包。
- **建议**：demo 阶段**单仓多路由**即可；若后续两站域名与部署完全分离，再拆仓或拆应用。

---

## 五、稳定性与专业性原则（落地方式）

- **依赖版本**：核心框架与库采用**当前稳定版**（如 Next.js 14.x、React 19.x），避免盲目追新；升级前在分支验证。
- **类型与测试**：TypeScript 严格模式；关键路径与组件有单元/集成测试（如 Vitest、React Testing Library），保证重构与交接安全。
- **性能**：图表按需加载（ECharts 可懒加载）；大列表考虑虚拟滚动；图片使用 Next.js Image；Lighthouse 与 Core Web Vitals 达标。
- **无障碍**：shadcn/ui 与 Radix 已做基础无障碍；表单与交互需保证焦点、键盘、ARIA；图标与图表需提供文字说明或 aria-label。
- **模型与 AI 辅助**：在自动模式下使用模型时，**选用适合任务的模型**（如代码生成、设计建议、文案润色），**不以省费用为首要**；可调用 skill、采用先进但稳定的工具链。

## 六、与 MRD、竞品调研的对应关系

| MRD/调研项 | 本方案对应 |
|------------|------------|
| UI/UX 保持「机构、严谨、深色系/科技感」 | shadcn/ui + 深色 token + Lucide/Heroicons；Next.js 稳定版。 |

| 风控数据须经后端校验 | 前端仅展示；接口与权限由后端控制；本方案不涉及后端技术选型。 |
