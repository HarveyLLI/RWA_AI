# AI 在业务模式中的角色与环节

> 整理日期：2026-02-26  
> 用途：在既有业务链条下，厘清 AI 能做什么、在哪些环节起作用、是对内还是对客户的支持，供产品与 demo 设计参考。

---

## 一、业务链条简要回顾

1. **借贷端**：客户有借贷需求 → 我们产生借贷订单（利率、期限等）→ 放款需要资金。
2. **债的转化**：借贷需求通过合约变成「债」的 DeFi 产品。
3. **资金端**：N 个投资者在基金会平台上买债获收益 → 募资所得作为放款资金。
4. **合规**：我们提供合约与平台（技术+风控），基金会面向投资者募资与卖债。
5. **双端呈现**：官网展示技术能力、合作方、透明度；基金会平台做产品发布与链上募资。

---

## 二、AI 在哪些环节能起作用？（按链条）

### 1. 借贷端（资产产生）

| 环节 | AI 能做什么 | 对内 / 对客户 |
|------|-------------|----------------|
| **尽调与定价** | 辅助评估抵押品质量、建议 LTV 上限、利率与期限区间（基于历史与市场） | **对内**：提升审批效率、定价一致性 |
| **合同与结构** | 辅助生成或校验合同条款、债的结构（期限、还款方式等） | **对内**：减少人工差错、加快上线 |
| **客户沟通** | 回答借款人关于产品、资格、流程、利率的常见问题；引导填表或补材料 | **对客户**：更好的借款端体验与自助服务 |

### 2. 债的发行与资金池管理

| 环节 | AI 能做什么 | 对内 / 对客户 |
|------|-------------|----------------|
| **池子设计** | 根据资产特征与监管约束，辅助设计池子结构、披露口径 | **对内**：产品设计支持 |
| **实时风控** | 持续监控 LTV、抵押品价值、市场波动；接近清算线或异常时告警 | **对内**：运营与风控；**对客户**：若把「预警」开放给借款人或合作方，即对客户的支持 |
| **合规与审计** | 异常交易检测、KYC/AML 辅助筛查、审计日志摘要 | **对内**：合规与审计效率 |

### 3. 资金端（投资者与募资）

| 环节 | AI 能做什么 | 对内 / 对客户 |
|------|-------------|----------------|
| **投资者理解产品** | 用自然语言解释池子风险、APY、条款、托管证明含义（合规表述：Monitor / Verify，避免 Invest / Earn） | **对客户**：降低理解门槛、增强信任 |
| **投资者监控与决策** | 投资者通过自己的 **AI Agent** 调用我们的 API：实时看 APY、LTV、NAV；设置提醒（如 LTV 超阈值）；在合规与授权前提下，由 Agent 执行「申购/下单」等 | **对客户**：更好的投资端体验与效率，我们提供数据与接口 |
| **投资者服务** | 智能客服：常见问题、账户与持仓查询、如何验证资产安全 | **对客户**：减轻人工客服、7×24 响应 |

### 4. 合作方与生态（DApp、基金会、开发者）

| 环节 | AI 能做什么 | 对内 / 对客户 |
|------|-------------|----------------|
| **DApp / 基金会接入** | 文档与示例的智能问答；根据其技术栈推荐接入方式（API/MCP）；辅助排查接口问题 | **对客户**：更快对接、更低接入成本。*注：基金会多为线下 KYC，官网「开放接入」用途有限，见《项目一AI用途纠正与老板思路及AI推广》。* |
| **开发者 / AI Agent 构建者** | 我们提供 API、MCP、Key、文档与示例代码；他们用我们的数据与风控能力做自己的 Agent 或应用 → 我们支持的是**开发者**，他们再服务其终端用户 | **对客户**：对「开发者」这一层客户的支持，间接支持更多终端用户 |
| **合作方运营** | 我们内部用 AI 分析各池子表现、投资者分布、异常行为，优化产品与风控策略 | **对内**：运营与策略支持 |

### 5. 产品推广（之前遗漏，已补充）

| 环节 | AI 能做什么 | 对内 / 对客户 |
|------|-------------|----------------|
| **推广文案与内容** | 用 AI 写推广文案、落地页、产品与能力介绍、合规话术（避免 Invest/Deposit/Earn）；社媒、邮件、投教内容；多语言推广素材（中/英等） | **对内**：营销与运营工具，提升推广效率；**对外**：更多人与 Agent 发现我们的产品，与「Agent 找收益会找到我们」形成闭环 |
| **被 Agent 发现** | 官网与平台对 Agent 友好（结构化数据、可读的 APY/风险/透明度），让「找收益」的 Agent 能发现、读懂、推荐我们 | **对客户**：投资者的 Agent 找赚钱项目时把我们算进去并推荐/下单 |

## 三、一句话归纳：对内 vs 对客户

- **对内**：AI 在**尽调与定价、合同与结构、风控监控、合规与审计、池子设计、运营分析**等环节起作用，主要带来**效率、一致性、可扩展性**，并降低操作与合规风险；**产品推广**（AI 写文案、做内容）提升获客与「被 Agent 发现」的效率。
- **对客户**：  
  - **借款人**：产品与流程说明、自助查询（如抵押率、建议操作）、申请引导。  
- **投资者**：产品与风险解释、通过 **AI Agent** 监控与告警、在合规前提下由 Agent 执行操作、智能客服；**被 Agent 发现**（官网与平台对 Agent 友好，让「找收益」的 Agent 能发现、读懂、推荐我们，见《项目一AI用途纠正与老板思路及AI推广》）。
  - **合作方 / 开发者**：接入与调试支持、API/MCP/Key 与文档，让他们能更好地服务其用户。

即：**对内 = 更好支持我们的运营、风控、合规与推广；对客户 = 更好支持借款人、投资者、合作方与开发者，并让投资者的 Agent 能发现与使用我们**，两端都可以做，且能形成「我们内部用 AI 管好资产与风控、用 AI 做推广 → 对外对 Agent 友好、把透明与能力开放给客户与 Agent」的闭环。

---

## 四、对产品与 demo 的启示

1. **对外宣传**：可强调「AI 赋能机构级 RWA」— 既包括我们**内部**用 AI 做风控与运营，也包括**客户**用 AI Agent 监控、验证与在合规范围内行动。
2. **官网 / 新导航**：技术能力介绍里可有一块「AI 与风控」「AI 与透明度」（我们如何用 AI 保证资产真实、风险可控）；再有一块「为开发者与 AI Agent 提供接口」（对客户的支持）。
3. **基金会平台**：投资者端可设计「用你的 AI Agent 连接我们」的入口（API Key、MCP、文档），以及人类可读的「产品与风险说明」（可由 AI 生成或辅助）；平台对 Agent 友好，让「找收益」的 Agent 能发现并推荐我们。
4. **AI 推广**：用 AI 写官网/平台推广文案、产品与能力介绍、合规话术、多语言素材，把新产品推广出去，让更多人与 Agent 发现我们。
5. **合规**：所有对投资者的 AI 能力表述保持「Monitor / Verify / Technology」，避免「Invest / Earn」；若涉及交易，明确「在用户授权与合规前提下」。
