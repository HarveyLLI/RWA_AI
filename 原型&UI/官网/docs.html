Quick Start

Get started in three steps—connect to our API and start querying pool data in minutes.

Integration in 3 steps

1Request an API key from the developer dashboard or contact your integration manager.
2Set the header X-API-Key: your_key on every request to https://api.antalpha.com/v1/
3Call GET /pools to list all active pools and their current APY.

Authentication

All API requests require an API key passed in the request header. MCP tools use the same key for server-to-server auth.

API Key (Header)

Include X-API-Key: ap_live_xxxxxxxxxxxx in every request. Keys are scoped to read-only by default.

Example request
123
curl https://api.antalpha.com/v1/pools \
  -H "X-API-Key: ap_live_xxxxxxxxxxxx" \
  -H "Accept: application/json"

MCP (Model Context Protocol)

Connect AI agents to Antalpha Prime data via MCP. Agents can discover products, monitor risk, and verify custody—all through standardized tool calls.

Available Tools

get_pool_list Read

Returns all active lending pools with APY, TVL, maturity, and risk grade. Use for opportunity discovery.

get_pool_list(status?: "active" | "matured" | "all") → Pool[] 
{
  "tool": "get_pool_list",
  "result": [
    { "id": "pool_btc_earn_01", "name": "BTC-Backed Earn", "apy": 0.085 }
  ]
}
get_pool_apy Read

Get current APY and yield curve for a specific pool. Agents use this to compare returns.

get_pool_apy(pool_id: string) → { apy: number, nav: number, maturity: string }
get_realtime_ltv Read

Real-time LTV and risk status across pools. Agents use this to monitor health and alert on thresholds.

get_realtime_ltv(pool_id?: string) → { global_ltv: number, pools: RiskEntry[] }
get_custody_proof Read

Verify custody attestation and cold wallet balances for a pool's BTC collateral.

get_custody_proof(pool_id: string) → { custodian: string, wallets: Wallet[], hash: string }
verify_kyc_status Read

Check the KYC/AML status of a wallet address. Returns verification level and expiry.

verify_kyc_status(address: string) → { verified: boolean, level: string, expires: string }

{
  "mcpServers": {

      "url": "https://mcp.antalpha.com/sse",
      "transport": "sse",
      "headers": {
        "X-API-Key": "ap_live_xxxxxxxxxxxx"
      },
      "description": "Antalpha Prime RWA data — pools, risk, custody"
    }
  }
}

MCP Quick Start

1Copy the config above into your agent's MCP server configuration file.
2Replace ap_live_xxxxxxxxxxxx with your API key.
3Your agent can now call get_pool_list, get_realtime_ltv, etc.
